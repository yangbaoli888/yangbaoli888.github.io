Java容器类常考知识点

Jul 18, 2019秋招点击

1、ArrayList、Vector与LinkedList有什么区别

ArrayList，Vector，LinkedList类均在java.util包中，均为可伸缩数组，可以动态改变大小的数组。

  1. ArrayList与Vector是基于Object[] array来实现的，支持用下标进行访问元素，而LinkedList是基于双向链表实现的，不支持随机查找只能从头遍历，但是增加删除元素不用移动已有原素。

  2. ArrayList与Vector都有一个初始大小，当初始值不够时它们的扩展方式不同，Vector默认为扩展2倍（可以自定义扩展大小），ArrayList默认是扩展1.5倍（不可以自定义扩展大小）。

  3. ArrayList不是线程安全的，而Vector是线程安全的，所以Vector的效率要低于ArrayList，LinkedList也是非线程安全的。

2、HashMap、Hashtable与TreeMap的区别

  1. 继承的父类不同：HashMap继承自AbstractMap而Hashtable继承自Dictionary类。二者都实现了Map接口。
  2. 线程安全性不同：HashMap是非线程安全的，Hashtable是线程安全的。
  3. 是否提供contains方法：HashMap去掉了contains方法将其改成了containsKey，containsValue两个函数，而Hashtable保留了contains方法并且也提供containsKey与ContainsValue方法。
  4. Key和Value是否允许有null值：HashMap中只允许出现一个为null的Key，但是可以有多个为null的Value，Hashtable中不允许出现null值。
  5. 遍历方式的内部实现不同：HashMap与Hashtable都使用了Iterator作为迭代器，Hashtable还使用了Enumeration。
  6. hash值不同：HashMap重新计算Key的hash值，Hashtable直接使用Key对象的hash值。
  7. 内部实现使用的数组初始化容量与扩容方式不同：HashMap的默认容量是16并要求扩容后的容量一定是2的整数次幂，而Hashtable是11不要求扩容后的容量是2的整数次幂。HashMap扩容一次增加2 * old，而Hashtable则为2 * old + 1。
  8. 计算index的方式不同：HashMap使用(n - 1) & hash(key)方式计算index，其中n为数组长度。而Hashtable使用（hash(key) & 0x7FFFFFFF） % length。

3、使用自定义类型作为HashMap或Hashtable的Key需要注意哪些问题

  1. 如果想根据对象的相关属性来自定义对象是否相等的逻辑，此时就需要重写类的equals方法，重写equals方法就要重写hashCode方法。

  2. 当自定义类的多项作为HashMap（Hashtable）的Key时最好将类设计成不可变类。

    两个对象相等 ====>> hash值相等 hash值相等 ===No==>对象相等

4、向HashMap中插入元素的步骤是什么

  1. 调用Key的hashCode()方法生成hash值h1，如果h1在HashMap中不存在则直接添加新的映射关系。
  2. 如果HashMap中已经存在h1，找到h1所在链表的表头然后分别调用key的equals()方法比较新添加的Value与拥有相同h1的Value是否相同。
  3. 如果相同equals()返回true，新的映射关系会覆盖掉旧的；如果不同则将新的映射关系添加到链表中。

5、ConcurrentHashMap的实现原理

  1. 底层使用数组+链表实现，这点与HashMap与Hashtable类似；
  2. 底层把数据分为多个Segment，每个Segment都拥有一把锁，通过使用锁分离技术实现多线程安全，这点与Hashtable把整张表锁起来的方式不同，效率更高，默认是16倍。读操作不加锁；
  3. 有些方法需要跨Segment，比如size()和containsValue()方法它们可能要求锁定整张表而不是某些段，实现的方式是通过顺序锁定所有Segment，操作完毕后再按照顺序释放所有段的锁；
  4. 扩容：当段内的元素个数超过段容量的75%时就会触发扩容操作，扩容只是在段内扩容，在新元素插入前进行扩容避免了无效扩容（扩容后无新元素插入）。
