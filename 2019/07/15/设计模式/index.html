设计模式

Jul 15, 2019秋招点击

1.单例模式

 在某些情况下，有些对象只需要一个就行了，即每个类只需要有一个实例。比如：一个班只需要有一个班主任；Windows系统是多进程多线程的当访问文件时就不可避免的可能出现多个进程或者线程同时访问一个文件的情况，所以文件的处理必须通过单例模式来进行。单例模式确保一个类只有一个实例并且自行实例化该实例，提供一个可以访问实例的方法。
实现方法：

懒汉式： 懒汉式实现方法是在需要实例时才会创建一个。非线程安全，可以在getInstance方法加上synchronize实现线程安全，但是效率降低。

饿汉式： 在类装载时就实例化一个实例，线程安全，是一种比较常用的方法。

2.工厂模式
  1. 工厂模式
     工厂模式（Factory Pattern）是 Java 中最常用的设计模式之一。这种类型的设计模式属于创建型模式，它提供了一种创建对象的最佳方式。
     在工厂模式中，我们在创建对象时不会对客户端暴露创建逻辑，并且是通过使用一个共同的接口来指向新创建的对象。
    角色： 抽象产品接口、具体产品类、工厂方法、客户端

     抽象工厂模式是在工厂模式的基础上增加了一个抽象工厂类，把工厂当做是产品来对待。
